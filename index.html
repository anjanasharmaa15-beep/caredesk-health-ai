How CareDesk works
CareDesk is an AI-powered front-desk assistant for small health practices. It handles routine patient questions so your staff can focus on care — not call volume.
🧩 System prompt as configuration Architecture
Each practice type (dental, GP, physio, pediatric) runs on a different system prompt. Swapping the clinical context takes minutes. No retraining, no new model — just a different instruction set. This is the core of how the product stays flexible across verticals.
📋 Intake-first conversation design Product
Before the chat starts, the patient provides their name, visit type, and reason. This pre-populates the first message to Claude, giving the model context without requiring the patient to repeat themselves. Better inputs → better outputs from message one.
⚠️ Escalation detection Safety
Every bot response is scanned for clinical trigger words (diagnosis, prescription, severe pain, emergency, 911, difficulty breathing). When detected, the message is flagged in the sidebar escalation log and a visual indicator is shown to the patient. This creates an auditable trail for the clinic — and keeps the AI's lane clearly defined.
📤 Structured handoff summary Workflow
At the end of every session, CareDesk generates a structured summary: patient name, visit type, topics discussed, and any escalation flags. The clinic receives this via email (in production: integrated into the PMS). This closes the loop between AI-assisted intake and human-led care.
Production roadmap
1
Backend API proxy — move the API key server-side so it's never exposed in the browser.
2
Scheduling API integration — connect to Jane App, Calendly, or the clinic's PMS so patients can actually book, not just ask about booking.
3
Auth + patient history — returning patients get context-aware responses based on their record (read-only EHR access).
4
Escalation workflow — flagged conversations routed to a staff dashboard with one-click callback or SMS to patient.
5
Analytics layer — track top questions by practice type, escalation rate, session resolution rate. Feed back into prompt improvement.
