How CareDesk works
AI Agent, not a chatbot
CareDesk v2 uses Claude's tool-use API — it reasons, picks tools, executes them, and loops until it has a complete answer for the patient.
🔁 Agent loop Architecture
Every message runs through a tool-use loop: Claude decides which tool to call, executes it via the FastAPI backend, incorporates the result, and reasons again — until it reaches a final answer.
🩺 Clinical Triage Junior Doctor
A separate agent mode with a 9-step consultation framework: chief complaint → history → red flags → system review → differential diagnosis → urgency → next steps → investigations → safety net.
📅 Real scheduling Cal.com
When Cal.com is configured, the agent checks live availability and creates real bookings. Without it, mock mode returns realistic demo slots so the full flow works in development.
⚠️ Smart escalation Safety
Claude decides when to escalate — not keyword matching. It calls the escalate_to_staff tool with urgency levels (low → emergency) and can send email alerts for high-priority cases.
💾 Session storage SQLite
All sessions, messages, tool calls, and escalations are stored in SQLite — free, embedded, survives restarts. Summary is AI-generated by Claude, not a template.
